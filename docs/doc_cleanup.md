\# Documentation Cleanup Rules



\## Purpose

Convert messy, outdated notes into a clean, current, authoritative state.



\## Rules



1\. Only keep information that reflects the CURRENT implementation or intended design.

2\. Remove or relocate outdated ideas to ideas\_backlog.md.

3\. Resolve contradictions by choosing the most recent or explicitly stated behavior.

4\. Do NOT invent new features.

5\. Be concise and structured.



\## Output Format



\- current\_state.md → high-level summary (max \~500-1000 tokens)

\- player.md → only player-related systems

\- enemies.md → only enemy systems

\- systems.md → shared systems (bullets, collisions, scoring)



\## Style



\- Use headers (#, ##, ###)

\- Use bullet points

\- No paragraphs longer than 3–4 lines

\- Prefer explicit values (numbers, rules)



\## Critical Rule



If information is unclear or conflicting:

→ flag it instead of guessing
